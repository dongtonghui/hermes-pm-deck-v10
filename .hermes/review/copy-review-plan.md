## Review Persona
- **角色**：Steven（产品总监，工作坊主讲）
- **语气**：直接、实战、少黑话、像和同事聊天

- **禁忌**：不堆砌概念、不制造焦虑、不用「赋能/闭环/抓手」等黑话

- 自然度：是否像人写的，而不是 AI 生成的

### 句式问题
1. **过度正式**："构建完整自动化闭环" → "让它一直自动跑下去"
2. **概念堆砌**："用多 Agent 模板进行结构化分析" → "用几个 Agent 一起把这件事分析清楚"
3. **缺乏具体**："AI 工作流落地实践" → "怎么把 AI 真正用进工作里"
4. **过度总结**："方法论适用于任何入口形态" → "这个方法在哪儿都能用"

## 优先修改清单

1. 标题/副标题中去掉「深度」「落地」「范式」等词
2. 把「自动化」改成「自动运行」或直接用具体动作（如定时跑、自动推）
3. 把「闭环」改成「完整循环」或具体描述
4. 把「痛点」改成「真正的问题/麻烦」
5. 把「方法论」改成「方法」
6. 减少「场景」的重复，用「情况」「实际工作」等替换
7. 把「沉淀」改成「积累」
8. 把「迭代」改成「持续优化」或「慢慢改」
