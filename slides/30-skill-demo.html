# hardware-pm-requirement-analysis/SKILL.md
---
name: hardware-pm-requirement-analysis
description: 用于硬件产品 PRD 前的多 Agent 需求分析
---

## 何时使用
写 PRD 之前，需要对需求做系统性分析，团队对一个功能是否该做存在争议时。

## 输入
1. 产品名称与一句话描述
2. 目标用户画像
3. 产品目标与约束
4. 需要分析的功能或问题

## 工作流
1. 并行生成 4 个 Agent 的分析
2. 让 4 个 Agent 互相交叉校验
3. 输出带证据的需求分析结论

## 输出格式
结构化 Markdown 报告，包含多视角分析、交叉校验、争议点、优先级建议、不确定项。
