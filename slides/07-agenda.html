今天会讲什么

60 分钟，4 条原则。现场讲授顺序为 1 → 3 → 2 → 4，因为植物灯案例会自然带出多 Agent 交叉校验。

01

给目标，不要只给任务

用植物生长照射仪的「氛围灯功能」案例，演示如何把"帮我写 PRD"改成"判断这个功能是否值得做"。

02

喂真实数据，不要空口问

用 Reddit 等社区真实数据，展示如何抓取海外用户声音，并把它变成可执行的产品洞察。

03

多角色互相挑毛病，不要单点信任

用 4 个 Agent 分别从用户、技术、商业、竞品视角交叉校验，证明氛围灯大概率是伪需求。

04

封装 + 自动化，不要每次都从零写

把 Reddit 情报抓取和多 Agent 分析封装成 Skill，加上定时任务和消息推送，构建完整自动化闭环。
