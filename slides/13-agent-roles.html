Roles

4 个 Agent 的角色定义

每个 Agent 只从一个视角分析，避免互相干扰；Prompt 要明确角色、目标、约束、输出格式。

U

用户视角 Agent

用户是谁？真正痛点是什么？使用场景有哪些？氛围灯是刚需吗？

T

技术视角 Agent

实现成本？对主功能影响？散热、寿命、能耗、认证如何变化？

B

商业视角 Agent

定价空间？溢价能力？市场容量？渠道接受度？ROI 如何？

C

竞品视角 Agent

竞品怎么做？为什么没做氛围灯？差异化机会真的成立吗？

Prompt 设计原则

  • 给角色、给目标、给约束、给输出格式
  • 要求引用证据，标注"事实 / 推测 / 待验证"
  • 明确要求 Agent 对其他角色的结论提出质疑
  • 使用同一套输入，避免信息不一致
