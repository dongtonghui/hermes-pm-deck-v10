# 硬件产品多 Agent 需求分析模板

## 使用说明

本模板用于硬件产品 PRD 前的多 Agent 需求分析。通过用户、技术、商业、竞品四个视角的独立分析和交叉校验，降低 AI 幻觉，提升分析严谨性。

---

## 第一步：明确产品目标与约束

在让 AI 分析之前，先填写以下信息：

| 字段 | 内容 |
|------|------|
| 产品名称 | |

| 时间约束 | |
| 技术边界 | |

## 第二步：四个 Agent 的 Prompt

### Agent 1：用户视角

```
你是一位用户研究员，专注于[产品领域]。

产品背景：
- 产品名称：[填写]
- 目标用户：[填写]
- 需要分析的功能/问题：[填写]

请从用户视角分析：
1. 目标用户是谁？有哪些细分人群？
2. 他们在什么场景下使用这个产品？
3. 这个功能是否解决了真实痛点？
4. 用户是否愿意为这个功能付费？
5. 列出支持观点的 evidence 和反对观点的 evidence

输出要求：
- 用 bullet points 列出
- 每个结论标注是"事实"还是"推测"
- 列出 3 个最需要验证的假设
```

### Agent 2：技术视角

```
你是一位硬件工程师，专注于[产品领域]。

请从技术视角分析[功能/问题]的影响：
1. 硬件成本增加多少？
2. 对散热、寿命、能耗的影响？
3. 对主功能是否有干扰？
4. 结构设计和供应链复杂度变化？
5. 认证（安规、EMC）是否会受影响？

输出要求：
- 量化分析尽量给出估算范围
- 标注不确定项
- 给出 "做" 和 "不做" 的风险对比
```

### Agent 3：商业视角

```
你是一位商业分析师，专注于消费硬件产品。

请从商业视角分析[功能/问题]：
1. 市场容量和增长趋势
2. 目标价格带和用户价格敏感度
3. 这个功能能支撑多少溢价？
4. 渠道（电商 / 线下）是否接受？
5. ROI 估算

输出要求：
- 给出清晰的商业判断
- 标注数据来源或假设
- 列出关键风险
```

### Agent 4：竞品视角

```
你是一位竞品分析师，专注于[产品领域]。

请分析以下竞品的做法：
- [竞品 A]
- [竞品 B]
- [竞品 C]

分析维度：
1. 它们是否提供了类似功能？
2. 它们的核心卖点是什么？
3. 为什么它们做了（或没做）这个功能？
4. 我们的差异化机会在哪里？

输出要求：
- 用表格对比
- 列出核心洞察
- 标注信息来源
```

---

## 第三步：交叉校验 Prompt

```
你是一位资深产品评审，现在有四份分析报告（用户/技术/商业/竞品）。

请完成以下任务：
1. 找出四份报告中的矛盾点
2. 找出每份报告中可能存在的证据不足或逻辑漏洞
3. 补充被遗漏的重要视角
4. 给出最终判断：[功能/问题]是否值得优先投入？

输出格式：
- 矛盾点：...
- 漏洞：...
- 遗漏：...
- 最终建议：...
- 下一步验证：...
```

---

## 使用原则

1. **角色分离**：每个 Agent 只从一个视角分析
2. **证据驱动**：每个结论都要有来源，否则标注为待验证
3. **人在回路**：最终决策必须由人做，AI 只提供输入
